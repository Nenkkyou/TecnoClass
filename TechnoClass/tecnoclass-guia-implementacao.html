TecnoClass - Guia de Implementação

Este documento fornece instruções para implementar o novo design do TecnoClass, um aplicativo educacional moderno e responsivo.

Arquivos Criados

Os seguintes arquivos foram criados para o redesign do TecnoClass:

  • tecnoclass-styles.css - Estilos CSS modernos e responsivos
  • tecnoclass-index.html - Estrutura HTML semântica
  • tecnoclass-interactions.js - Interações e microanimações
  • service-worker.js - Funcionalidades PWA e offline
  • manifest.json - Configuração para instalação como PWA

Passos para Implementação

  1. Preparar a Estrutura de Pastas

    Crie as pastas conforme a estrutura recomendada acima.

  2. Adicionar Imagens

    Você precisará criar ou obter as seguintes imagens:

    • Logo do TecnoClass (formato SVG recomendado)
    • Ícones para PWA (192x192 e 512x512)
    • Imagens para os cards de cursos

  3. Ajustar Caminhos nos Arquivos

    Após organizar os arquivos nas pastas, atualize os caminhos nos arquivos HTML, CSS e JS para refletir a nova estrutura.

    Por exemplo, no HTML:

    <link rel="stylesheet" href="css/styles.css">
<script src="js/interactions.js"></script>

  4. Testar Localmente

    Abra o arquivo index.html em um navegador para testar o funcionamento básico.

  5. Fazer Upload para o GitHub

    Faça upload de todos os arquivos para seu repositório GitHub, mantendo a estrutura de pastas.

Recursos Adicionais Necessários

  • Fontes

    O design utiliza as fontes Poppins, Inter e Nunito do Google Fonts. Elas já estão incluídas no HTML via CDN.

  • Imagens

    Você precisará criar ou obter imagens para:

    • Logo do TecnoClass
    • Cards de cursos (programação, cybersegurança, IA, product owner)
    • Ícones para PWA

Nota Importante

Para que o GitHub Pages funcione corretamente com esta nova estrutura:

  1. Certifique-se de que o arquivo index.html esteja na raiz do repositório
  2. Verifique se todos os caminhos de arquivos estão corretos (relativos à raiz)
  3. Se estiver usando rotas no aplicativo, pode ser necessário configurar um arquivo 404.html para redirecionamento

Características do Novo Design

  • Design Responsivo - Funciona em dispositivos móveis, tablets e desktops
  • Modo Claro/Escuro - Alternância suave entre temas
  • Microinterações - Animações sutis para melhorar a experiência do usuário
  • Acessibilidade - Contraste adequado, navegação por teclado e estrutura semântica
  • PWA - Funcionalidade offline e instalação como aplicativo

Personalização

Você pode personalizar facilmente o design ajustando as variáveis CSS no início do arquivo styles.css:

:root {
  /* Paleta de cores - Modo Claro */
  --primary-color: #3a86ff;
  --secondary-color: #4cc9f0;
  --accent-color: #4895ef;
  /* ... outras variáveis ... */
}

Para adicionar novos componentes ou páginas, siga a estrutura HTML existente e utilize as classes CSS já definidas para manter a consistência visual.
